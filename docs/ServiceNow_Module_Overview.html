Feature Summary

Incident Management

  • Live incident list with priority, state, and search filtering
  • CRUD operations — create, update, close (two-step resolve/close)
  • Server-side pagination via SNOW Table API sysparm_limit/offset
  • Test incidents — create and close test data from the UI
  • Assignment group filtering — scope incidents to specific teams

Auto-Acknowledge

  • Background poller — configurable frequency (1–60 min)
  • Auto-transition — New (state=1) to In Progress (state=2)
  • DB-only dedup — stateless, safe for K8s pod restarts
  • Manual trigger + historical log viewer with search

SLA Compliance Reporting

  • Resolution SLA — business-hour-aware expected close times
  • Response SLA — configurable response column mapping
  • Per-priority thresholds — P1: 4h, P2: 8h, P3: 24h, P4: 72h
  • Date range filters — Daily, Weekly, Monthly, Custom
  • PDF export for all SLA reports

Analytics & Dashboards

  • Live stats — total, open, critical, resolved today
  • Incident analytics — by priority, state, category
  • RITM reports — catalog item breakdown
  • DataTable grids — sort, paginate, column reorder, search

Deployment & Operations

Hot-Drop Deployment

# 1. Build the module
npm run build:module -- servicenow

# 2. Output
dist-modules/servicenow/manifest.js

# 3. Enable via Module Manager UI
POST /api/modules/servicenow/enable

No platform rebuild. No server restart. The Module Manager UI handles enable/disable lifecycle.

Database Auto-Provisioning

TablePurpose
sn_module_configKey-value config store
sn_sla_thresholdsSLA targets per priority
sn_business_hoursBusiness hours per day
sn_auto_acknowledge_logAuto-ack execution log

Tables are auto-created from Schema.json on first enable. Seed data from DefaultData.json.

Dev Mode

Vite dev server loads source manifest.jsx directly. Full HMR + alias resolution.

Prod Mode

Self-contained ES module bundle. Loaded via dynamic import() with cache-busting.

K8s Ready

Stateless API, DB-only dedup, no in-memory state. Safe for multi-pod horizontal scaling.

Quality & Engineering Standards

Code Quality

  • Zero hardcoded strings — All text in JSON config files
  • Comprehensive file headers — Purpose, usage, architecture in every file
  • Structured logging — Winston + DB persistence at all levels
  • Dead code cleanup — Unused components flagged as deprecated
  • Reusable DataTable — Single component for all grids across module

Security & Performance

  • Password redaction — Connection config never exposes credentials
  • Basic Auth over HTTPS — ServiceNow API authentication
  • Request timeouts — 15s timeout on all SNOW API calls
  • Field projection — Only requested columns fetched (sysparm_fields)
  • Pagination — Server-side limit/offset for all list endpoints

Module Contract

  • manifest.jsx — Self-describing entry point with navItems, getViews(), getConfigTabs()
  • constants.json — Module metadata (id, name, version, description)
  • Schema.json — Database table definitions (auto-provisioned)
  • Lifecycle hooks — onEnable() / onDisable() for startup/teardown

Timezone Handling

  • UTC parsing — SNOW dates always parsed as UTC (no 'Z' suffix)
  • IANA conversion — Intl.DateTimeFormat for display timezone
  • Business hours — SLA calculations use configured business hours
  • Auto-detect — Fetch timezone from SNOW instance properties
